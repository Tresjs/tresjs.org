It's been a while since you heard from us, and we've been busy working on new things.

### 🚀 Roadmap for 2026

This year we want to focus on a couple areas and tackle common issues as well as hand you more features to use with Tres.

- Rapier support - release a first major version of our physics package with support for Rapier physics engine
- Further improve support for WebGPU
- Better Developer Experience and more concise documentation
- Performance improvements and bug fixes
- Plugin API <placeholder to RFC discussion>
- Move to monorepo and modularize the ecosystem more


#### Rapier support

This has been the area where Tres still has a feature gap compared to what the Three.js and compatible plugins have to offer. 
We want to provide you with a package to handle physics the Tres way.


#### Improve DX and documentation

The switch from v4 to v5 involved a lot of changes and unfortunately one of the things that we lost are some of the translations. 
There are a lot of advantages and disadvantages that come with supporting localized documentation, which is why we stuck to English for v5. 
However, beside English we noticed a big chunk of people from the Chinese-speaking community which where very active around Tres v4. 

Therefore, we decided that we want to bring back a Chinese translation of at least our Tres Core docs. 
So if you speak Chinese or if you know someone that is interested in 3D and speaks Chinese and wants to contribute to 
the translation efforts of Tres, please hit us up in Discord!


#### 🔮Plugin API

Our very own Jaime created an RFC discussion which suggests the idea of introducing a plugin API to Tres. You can read about it here <placeholder>
Please let us know what you think about it.


#### Monorepo and modularization

As some of you, who contributed to Tres might have noticed, we switched to a monorepo structure for our codebase. 
This allows us to better manage our packages and dependencies, and also makes it easier for contributors to work on multiple packages at the same time.
Most of the old repositories are now archived and the new monorepo is available where you could find the core package so far <placeholder>.

### 🛠️ Developer Experience Improvements

In addition to the major changes, we plan several smaller improvements to enhance your developer experience.
